UFO API

The UFO API is a collection of UFO Sigthings served through an API.

The original dataset is coming from The National UFO Reporting Center. The data has been modified and augmented with location data, to allow location based searches. Note that the returned location data is in (lon, lat) format.
An example of a visualization using this data can be found here.

Queries

The API has three base queries. Each base query has a subset of query parameters, see the docs for more info.

Search

The search base query is used to only return UFO sightings that qualify a set of query parameters. Each query parameter is optional and can be shape, city, state or date.

Location Near

The location near base query is used to get UFO sigthings near a specific location, by supplying a center point and an optional radius to declare how big the area from which to returns sightings from should be.

Location Bounding Box

The location bounding box base query is used to get UFO sightings within bounding box. The box is defined by an upper right and a lower left corner.

Search /api/sightings/search

Search endpoint. The possible search parameters below are all optional and can be applied as needed.

Example: /api/sightings/search?from=200/5/5&to=2016/8/7&shape=triangle&city=new%20york&state=ny&limit=50&skip=0

from

Returns only sightings after this date (including).

Format: accepts any valid javascript Date string.

Example: from=2016/5/5

to

Returns only sightings before this date (excluding).

Format: accepts any valid javascript Date string.

Example: to=2016/8/7

shape

Returns only sightings with the requested shape.

Format: accepts any string (not case sensitive).

Example: shape=triangle

city

Returns only sightings with the requested city.

Format: accepts any string (not case sensitive).

Example: city=new york

state

Returns only sightings with the requested state.

Format: accepts any string (not case sensitive).

Example: state=NY

limit

Amount of sightings to return - default=100.

Format: number.

Example: limit=50

skip

Amount of sightings to skip.

Format: number.

Example: skip=50

Location - Near /api/sightings/location/near

Location near endpoint. This query takes in either lat/lon pair or a location, and returns all sightings within a specificed distance (radius) of the lat/lon. If no radius is supplied, all sightings will be returned, returning the closests sightings first.

Required: lat/lon or a location.

Example: /api/sightings/location/near?lat=40.7294285&lon=-73.9958957&radius=1000&limit=50&skip=0

lat

The latitude for the center of the search. Must be used together with longitude.

Format: number between -90.0 and 90.0.

Example: lat=40.7294285

lon

The longitude for the center of the search. Must be used together with latitude.

Format: number between -180.0 and 180.0.

Example: lon=-73.9958957

location

String representation of a location. Is translated into lat/lon pair using a Geocoder. If both lat/lon and location is specified, lat/lon pair is preferred.

Format: string, accepts anything Google accepts.

Example: location=New York,NY

radius

The radius defining the circle around the center location from which to include sightings from.

Format: number in meters.

Example: radius=1000

limit

Amount of sightings to return - default=100.

Format: number.

Example: limit=50

skip

Amount of sightings to skip.

Format: number.

Example: skip=50

Location - Bounding Box /api/sightings/location/bbox

Location bounding box endpoint. This query returns all sightings in the specified bounding box.

Required: bbox or a location.

Example: /api/sightings/location/bbox?bbox=-74.259,40.477,-73.700,40.917&limit=50&skip=0

bbox

The bounding box coordinates for which to return sightings within. Specified as the lower left corner and upper right corner.

Format: comma seperated numbers (lowLon, lowLat, upLon, upLat).

Example: bbox=-74.259,40.477,-73.700,40.917

location

String representation of a location. Is translated into a bounding box using a Geocoder. If both bbox and location is specified, bbox is preferred.

Format: string, accepts anything Google accepts.

Example: location=New York,NY

limit

Amount of sightings to return - default=100.

Format: number.

Example: limit=50

skip

Amount of sightings to skip.

Format: number.

Example: skip=50
